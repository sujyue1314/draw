设计决策

1. 类型文件拆分
将 ExecutorCtx 放在 commands.ts 而非单独文件。因为 ExecutorCtx 依赖 CanvasState 和 ConversationTurn， 使用 import('./canvas') 类型导入避免循环依赖，同时保持 commands.ts 作为 action 执行的唯一类型入口。
2. AspectRatio 类型为联合字符串
使用 '16:9' | '9:16' | '1:1' | '4:3' | '3:4' 而非枚举。字符串联合类型在 JSON 序列化时更自然， 与 LLM 返回的 ratio 格式一致，无需额外转换。
3. HistorySnapshot 包含 remoteImageUrl
同时保留 imageUrl（blob URL）和 remoteImageUrl（DashScope 原始 URL）。 blob URL 刷新后失效，remoteImageUrl 可用于恢复。PR 10 持久化时会用 remoteImageUrl。

偏差

1. MCP 与 Git 凭证不一致
MCP GitHub 工具认证为 vjrm300 账号，git CLI 凭证为 sujyue1314。 解决方案：git push 用 sujyue1314 凭证推送代码，MCP 工具用于查看仓库状态。
2. PR 1 直接推送到 main
规范要求每个 PR 都通过 Pull Request 合并。但仓库为空时无法创建对比分支，PR 1 作为初始化提交直接推送到 main。 从 PR 2 开始将严格走 feature branch → PR → merge 流程。

权衡取舍

TranscriptOverlay 使用条件渲染 vs CSS 隐藏
选择条件渲染（if (!text) return null），不渲染空 DOM。 优点：无多余 DOM 节点；缺点：text 变化时有挂载/卸载开销。 对于低频变化的文本浮层，条件渲染更简洁。
